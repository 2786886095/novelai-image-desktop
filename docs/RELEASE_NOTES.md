## Langbai NovelAI Studio

请根据你的操作系统下载对应安装包：

| 系统 | 安装包 | 安装说明 |
| --- | --- | --- |
| 🪟 **Windows** (x64) | `Langbai-NovelAI-Studio-0.9.0.exe` | 便携版，双击即用，无需安装 |
| 🍎 **macOS** (Intel + Apple 芯片通用) | `Langbai-NovelAI-Studio-0.9.0-universal.dmg` | 拖入「应用程序」；**未签名**，首次打开请右键 →「打开」 |
| 🍎 **macOS**（压缩包，同上通用版） | `Langbai-NovelAI-Studio-0.9.0.zip` | 解压后即为 `.app`，同样需右键「打开」 |
| 🐧 **Linux** (x64) | `Langbai-NovelAI-Studio-0.9.0.AppImage` | `chmod +x` 后直接运行 |
| 🤖 **Android** | `app-release.apk` | 直接安装；未签名，需允许「未知来源」 |
| 📱 **iOS** | `novelai-mobile-unsigned.ipa` | **未签名**，需用 AltStore / Sideloadly 等工具自行侧载 |

> 桌面端与移动端均为 **API-only** 客户端，需自备 NovelAI Persistent API Token。

### v0.9.0 更新内容

- **局部重绘**：撤回 / 重做改用箭头按钮（↶ / ↷），并把工具栏移到画布上方，绘制蒙版时不再被按钮遮挡。
- **漫画生成器精简为 4 步**：故事与参考图 → 全局设定 → 分镜 → 生成；参考图管理并入第 1 步，移除独立的参考图步骤、返回生图 / 载入示例 / 成人分支 / 上一帧延续等冗余项；批量生成改为逐张串行。
- **新增 AI 一致性检测**：对已转换的分镜英文提示词做跨镜一致性审校（角色 / 服装 / 场景 / 时间线），自动补齐并修正不一致处。
- **AI 反推默认模板**：内置默认替换为「Danbooru 标签｜支持反推范围选择」版（整张 / 角色 / 物品 / 场景）；「恢复默认」改为取真实内置默认，不再把已自定义内容当默认。
- **新增「记录」标签页**：集中查看每次反推 / 转换 / 拆分镜 / 一致性检测发送给 AI 的 system + user 内容与原始返回，可展开 / 复制 / 清空。
- **安全与健壮性修复**：
  - 导入漫画项目 JSON 时剥离外部文件路径，导出 ZIP 仅打包输出目录内、通过图片校验的文件，且改用相对图片名（不再泄露本机绝对路径）。
  - `openExternal` 仅放行 http/https；导入旧 / 残缺项目统一归一化不再崩；队列「自动导出 ZIP」不再漏掉刚生成的图片；本地缓存超额时降级保存并提示。
- **移动端**：修复反推 / 转换输入框光标跳动与 controller 泄漏；版本号同步至 0.9.0。

### v0.8.6 更新内容

- **局部重绘对标官网（不再出色块 / 不再不按提示词）**：蒙版导出前会**对齐到 64px 方块网格**（与官网可见的「方块蒙版」一致），并强制纯黑白二值化，消除半透明灰边导致的脏色块；画布预览同步显示对齐后的方块，所见即所发。重绘强度默认从 0.55 调到 **1.0**（遮罩区完全按提示词重画，越低越贴近原图），画笔默认 64px 对齐一个方块。
- **漫画生成器重做工作流（不再一坨）**：改为 **5 步向导**——① 导入故事 → ② 全局设定 → ③ 参考图 → ④ 分镜转换 → ⑤ 队列出图，每步只聚焦一件事；高级参数可折叠。
  - 新增**生成队列**：进度条 + 暂停 / 继续 / 取消 + 失败单张重试 + 缩略图墙；顺序生成时上一帧自动作为下一张的低强度氛围参考（即使 React 状态未刷新也能正确链式传递）。
  - 角色 / 场景 / 物品参考图可一键**反推并写入全局设定**，让每个分镜都带上人物设定。
  - 生成前显示单张 / 选中 / 全部三档预计 Anlas 与当前余额，并二次确认。

### v0.8.5 更新内容

- **三模式模版彻底不混用**：反推 / 转换现在严格按当前所选模式取模版（自定义覆盖 → 内置默认），**彻底忽略**旧的全局模版字段（`convertSystemPrompt` / `visionSystemPrompt`，无界面、易隐性覆盖当前模式），避免「选了自然语言却被旧模板带偏」。
- **输出格式自纠 + 清理**：请求时附带当前模式的明确输出约束；返回后清理 `Output:` / markdown / 引号 / 换行等包装；若输出明显跑偏（如自然语言模式返回了 tag 列表）会做**一次**低额度（900）修复请求。修复为「尽力而为」——即使启发式仍判定不符也不会硬失败丢弃结果。
- 新增 12 项测试覆盖模版取用、格式检测与清理。

### v0.8.4 更新内容

- **AI 反推 / 转换 报错更清晰 + 兼容推理模型**：实测确认请求格式、三模式模版、视觉反推在 xAI（grok-4.3 等）均正常工作并遵循模版。但默认模型 `gpt-4o` / `gpt-4o-mini` 在非 OpenAI 服务（如 xAI）不存在（返回 400），这是接入第三方服务时「返回为空 / 结果不对」的常见原因。本版：返回为空时明确提示要把模型填成该服务支持的名字（如 grok-4.3，可点检测模型选择），并区分推理模型的长度截断；同时把反推 / 转换输出额度调大（600/900 → 2000）。

### v0.8.3 更新内容

- **重绘预览与其他三个标签对齐**：局部重绘的预览原来用 grid + 视口高度上限 + 棋盘格卡片，显得更小、风格也不一致。现改为与生成/超分/后期同款（绝对定位 flex 居中、图片 `max-height:100%` 同样大小、阴影加在图片上、无卡片）。蒙版画布与图片同宽高比、同 max 约束并居中叠放，像素级对齐，绘制映射不受影响。

### v0.8.2 更新内容

- **彻底修复生成 / 超分 / 后期图片不完整**：经实测定位（画布=651、舞台=601、图片=878），根因是图片舞台用 `display:grid`，其隐式自适应行会被图片内容撑高，导致 `max-height:100%` 失效、竖图溢出被裁。改为 `flex` 居中 + 绝对定位内边距，并给工作区一个确定行高（`minmax(0,1fr)`）让画布有确定高度。复测：图片 601 = 舞台 601，完整不裁。
- **默认提示词模版不再隐藏 + 可一键恢复**：反推 / 转换的内置三模式模版（标签 / 自然语言 / 混合）现在直接显示在设置里，可随时编辑；每个模式带「↺ 恢复默认」按钮随时还原为内置模版；并标注「默认 / 已自定义」。三种模式各用独立系统提示词，**绝不混用**。
