# NovelAI 图像桌面控制台：基于 10 类开发 Skill 的成品级重设计蓝图

本文档把用户提供的 10 个研发 Skill 转换为本项目的可执行产品规范。目标不是堆功能按钮，而是让软件具备清晰的信息架构、稳定的后台网页自动化、可测试、可维护、可发布。

## 1. frontend-design：前端设计

- 采用“专业深色创作控制台”风格：高对比、低噪声、清晰层级。
- 核心动线固定为：登录状态 -> 提示词/图片输入 -> 功能参数 -> 成本读取 -> 执行生成 -> 自动保存/历史复用。
- 所有英文功能必须有中文解释：例如 Prompt=提示词、Anlas=积分、Vibe=风格参考。
- 复杂功能分区展示，不再把按钮堆在顶部：
  - 全局控制：Vibe、精确参考、多角色。
  - 主执行功能：文生图/图生图、局部重绘、增强、放大、导演工具。
  - 后处理/资产：历史、分组、设为基图、继续编辑。

## 2. cache-components：前端性能与缓存思想

- 当前项目不是 Next.js，但借鉴其“动态数据与可缓存数据分离”：
  - 动态数据：账号状态、积分、网页按钮成本、生成进度。
  - 可缓存数据：本地历史、预设、导出目录、功能文档矩阵。
- 账号/积分只从后台网页实时读取，不写死。
- 历史图片使用本地协议 `novelai-image://`，避免直接暴露文件系统路径给网页。

## 3. fullstack-developer：全栈边界

- Electron 主进程负责：
  - 登录会话、文件保存、图片协议、后台网页自动化。
- React 前端负责：
  - 中文交互、参数组织、资产管理、历史分组。
- 隐藏 NovelAI 官方网页负责：
  - 真实登录、真实扣费、真实生成。
- 不直接调用 NovelAI 图像 API，不保存密码。

## 4. frontend-code-review：前端代码审查

每次较大 UI 修改后检查：

- 组件是否单一职责。
- 是否存在用户看不懂的英文按钮。
- `useMemo` / 状态派生是否避免重复计算。
- 图片列表、历史列表是否有空态、错误态、加载态。
- 删除、清空、退出登录等高影响操作是否有明确文案。

## 5. code-reviewer：通用代码审查

每次发布前检查：

- Electron session 是否隔离。
- 后台网页脚本是否有超时保护。
- 文件删除/清理路径是否限制在项目或用户明确目录。
- 捕获失败是否返回可操作错误，而不是沉默失败。
- 打包产物是否能独立启动。

## 6. webapp-testing：网页应用测试

验收测试分三层：

- 静态构建：`npm run typecheck`、`npm run build`。
- 打包启动：`npm run pack` 后运行便携版 exe，确认主窗口可见。
- 交互测试：
  - 首次登录、Google 登录、重启免登录。
  - 彻底退出登录后不会自动登回。
  - 文生图/图生图/局部重绘/导演工具执行后自动捕获图片。

## 7. pr-creator：发布与变更说明

每次可交付版本都应记录：

- 用户可见变化。
- 修复的问题。
- 仍需人工验证的 NovelAI 网页流程。
- 打包文件路径与大小。
- 运行/回滚方式。

## 8. fix：格式与 lint 修复

- 当前项目使用 TypeScript 检查作为最低质量门槛。
- 发布前必须执行：
  - `npm run typecheck`
  - `npm run build`
  - `npm run pack`
- 后续可加入 ESLint/Prettier，统一格式和未使用变量检查。

## 9. update-docs：文档同步

代码变更需要同步：

- `README.md`：运行方式、打包方式、使用流程。
- 本文档：产品设计原则和验收清单。
- 功能矩阵：新增/删除 NovelAI 功能入口时必须更新。

## 10. find-skills：能力发现

当项目进入下一轮重构时，优先补齐：

- Electron 端到端测试能力。
- Playwright 对隐藏网页的稳定 selector 探测。
- UI 设计系统自动审计。
- 打包体积分析。
- 安全审计：会话隔离、敏感信息、文件访问边界。

## 当前优先级

1. 账号信息：稳定读取用户名、套餐、积分、账号 ID、存储位置。
2. 退出登录：彻底清除本地 session，避免自动登回。
3. 主动生成动线：左下角只执行当前主功能，全局参考作为附加条件。
4. 自动捕获：生成后自动保存，不要求用户手动点击。
5. 历史资产：支持删除、分组、复用、设为基图。
6. 局部重绘/导演工具：按官方网页真实流程做专用适配器。
